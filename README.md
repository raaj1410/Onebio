# OneBio AMR Atlas

A single browser-based and command-line tool for building a BioSample-anchored atlas that connects:

- genome assemblies
- RNA-seq runs
- AST phenotypes
- AMR gene calls
- gene-level expression
- locus-level and orthogroup-level analysis
- SQLite exploration and knowledge-graph outputs

The original project had the right pieces. This package wires them into one usable workflow instead of making users manually pass CSVs between scripts like it is 2009 and suffering is a dependency.

---

## What the tool does

### 1. Discovery

Runs the RNA-seq AMR checker to find candidate isolates with:

- genome availability
- transcriptome availability
- laboratory-derived AST phenotype evidence
- AMR phenotype source support from BV-BRC, NCBI AST Browser and EMBL-EBI AMR Portal

### 2. Manifest bridge

Converts checker output into strict downstream inputs:

```text
manifest.tsv
ast_long.tsv
checker_selected_rows.csv
manifest_qc.json
```

This is the main streamlining step. The checker output becomes a clean, reproducible input to all modules.

### 3. Genome module

Downloads genome FASTA + GFF, builds locus maps and optionally runs AMRFinderPlus.

### 4. RNA-seq module

Downloads or reuses FASTQs, performs QC, trimming, alignment, feature counting and TPM calculation.

FASTQ fallback order:

1. local FASTQ files if already available
2. ENA FASTQ URLs from the manifest
3. ENA lookup using the run accession
4. SRA Toolkit using `prefetch` and `fasterq-dump`

### 5. Atlas module

Builds a SQLite database with:

- `biosample`
- `locus`
- `expression`
- `ast`
- `amr_hits`
- `amr_expression` view
- `amr_summary_by_biosample` view

### 6. Extra analysis modules

Included utilities:

- Panaroo input preparation
- orthogroup-level expression aggregation
- same-gene/different-loci/multicopy expression catalog
- knowledge graph DOT generation
- low-mapping doctor mode

## Installation

### Recommended: Conda or Mamba

```bash
git clone <your-github-repo-url>
cd onebio_amr_atlas_tool
mamba env create -f environment.yml
mamba activate onebio-amr-atlas
```

## Fast demo

Generate small demo files:

```bash
onebio-atlas demo-data --out onebio_demo
```

Prepare a manifest from the demo checker output:

```bash
onebio-atlas prepare-manifest \
  --checker-table onebio_demo/demo/demo_checker_output.csv \
  --out onebio_demo/demo/manifest
```

The demo is intentionally tiny. It is for testing file contracts and UI behaviour, not for biological conclusions. Somehow this has to be said out loud.

### Optional local smoke test without SRA/ENA downloads

The demo also includes a toy genome and local FASTQ. This lets you test Module C onward without depending on public database downloads:

```bash
onebio-atlas rnaseq \
  --manifest onebio_demo/demo/manifest/manifest.tsv \
  --genomes onebio_demo/demo/toy_genomes \
  --fastq-root onebio_demo/demo/local_fastq \
  --out onebio_demo/toy_run/rnaseq \
  --threads 2 \
  --max-runs-per-biosample 1

onebio-atlas features \
  --manifest onebio_demo/demo/manifest/manifest.tsv \
  --genomes onebio_demo/demo/toy_genomes \
  --rnaseq onebio_demo/toy_run/rnaseq \
  --out onebio_demo/toy_run/features

onebio-atlas atlas \
  --manifest onebio_demo/demo/manifest/manifest.tsv \
  --genomes onebio_demo/demo/toy_genomes \
  --rnaseq onebio_demo/toy_run/rnaseq \
  --features onebio_demo/toy_run/features \
  --out onebio_demo/toy_run/atlas
```

This still needs Bowtie2, samtools and featureCounts installed. It avoids only the download circus.

## Full workflow from the command line

### Step 1: Discover candidates

```bash
onebio-atlas discover \
  --species "Klebsiella pneumoniae" \
  --out-root outputs/checker \
  --cache-root cache/checker
```

Optional NCBI AST Browser export:

```bash
onebio-atlas discover \
  --species "Klebsiella pneumoniae" \
  --out-root outputs/checker \
  --cache-root cache/checker \
  --ncbi-ast-export data/ncbi_ast_export.tsv
```

### Step 2: Convert checker output to module input

Use the combined RNA-seq-positive CSV generated by the checker:

```bash
onebio-atlas prepare-manifest \
  --checker-table outputs/checker/rnaseq_true_combined_<timestamp>.csv \
  --out work/manifest
```

Output:

```text
work/manifest/manifest.tsv
work/manifest/ast_long.tsv
work/manifest/checker_selected_rows.csv
work/manifest/manifest_qc.json
```

### Step 3: Run selected modules

For a safe test run, limit to one RNA-seq run per BioSample:

```bash
onebio-atlas run \
  --manifest work/manifest/manifest.tsv \
  --work-dir work/run1 \
  --steps genome,rnaseq,features,atlas,kg \
  --threads 8 \
  --max-runs-per-biosample 1 \
  --amr auto
```

Run only selected parts:

```bash
onebio-atlas run --manifest work/manifest/manifest.tsv --work-dir work/run1 --steps genome --threads 8
onebio-atlas run --manifest work/manifest/manifest.tsv --work-dir work/run1 --steps rnaseq --threads 8 --max-runs-per-biosample 1
onebio-atlas run --manifest work/manifest/manifest.tsv --work-dir work/run1 --steps features,atlas,kg --threads 8
```

## Important commands

### Query a gene in the atlas

```bash
onebio-atlas query-gene \
  --db work/run1/atlas/atlas.sqlite \
  --gene-like "%bla%" \
  --limit 20
```

### Diagnose low mapping

```bash
onebio-atlas doctor \
  --ref work/run1/genomes/SAMNXXXX/genome.fna \
  --fastq work/run1/rnaseq/fastq/SRRXXXX/SRRXXXX.fastq.gz \
  --out work/doctor/SRRXXXX \
  --threads 8
```

### Prepare Panaroo input files

```bash
python -m onebio_amr_atlas.prepare_panaroo_inputs \
  --genomes work/run1/genomes \
  --out work/panaroo
```

### Build orthogroup expression after Panaroo

```bash
python -m onebio_amr_atlas.build_orthogroup_expression \
  --panaroo panaroo_output/gene_presence_absence.csv \
  --run-index work/run1/rnaseq/rnaseq_run_index.tsv \
  --genomes work/run1/genomes \
  --out work/orthogroup_expression \
  --aggregate sum
```

### Same gene, different loci, different expression

```bash
python -m onebio_amr_atlas.same_gene_locus_catalog \
  --db work/run1/atlas/atlas.sqlite \
  --outdir work/same_gene_catalog \
  --expr-threshold 1 \
  --min-loci 2 \
  --min-biosamples 2 \
  --min-delta-expr 5
```

## Notes for non-bioinformatics users

Use the UI first. Choose:

1. `Discover candidate isolates`
2. `Build module manifest`
3. `Run pipeline modules`
4. `Explore SQLite atlas`

For the first real run, set:

```text
Max runs per BioSample = 1
AMRFinderPlus mode = auto
Steps = genome,rnaseq,features,atlas,kg
```

After that succeeds, increase the number of runs.

## Known limits

- Very large species searches can take a long time and produce large caches.
- SRA/ENA/NCBI availability changes. The resolver tries multiple routes, but public databases remain public databases, which is to say, occasionally haunted.
- AMRFinderPlus requires a local installation and database.
- Panaroo is not run automatically from the UI yet. This package prepares Panaroo inputs and consumes Panaroo outputs.
- The included demo is structural, not biological.


## Running the UI safely

Do not run `python src/onebio_amr_atlas/ui.py` directly. Install the package first:

```bash
pip install -e .
onebio-atlas ui
```

Alternative direct Streamlit launch:

```bash
PYTHONPATH=src streamlit run src/onebio_amr_atlas/ui.py
```


## Added UI modules in v0.2

The browser UI now includes the analysis pieces that should not have been left outside the main tool like abandoned lab notebooks:

- **Explore SQLite Atlas**: full multi-tab Streamlit atlas explorer with overview, filtered cohort, sample deep dive, antibiotic explorer, AMR gene explorer, gene/locus search, and same-gene/different-loci drilldown.
- **Orthogroup explorer**: reads Panaroo-derived orthogroup TPM, copy-number, mapping coverage, and AMR-linked orthogroup tables.
- **Knowledge graph viewer**: displays Module F `.dot` files directly in the browser using Graphviz.
- **Progress bars for module execution**: the UI now runs selected modules step-by-step with an overall progress bar and per-step progress indicator.

### Orthogroup workflow

First prepare Panaroo inputs from Module B genome folders:

```bash
onebio-atlas ui
# UI: Orthogroup and locus analyses -> Prepare Panaroo inputs
```

Run Panaroo externally on the prepared GFF files:

```bash
panaroo -i work/run1/panaroo/inputs/*.gff   -o work/run1/panaroo/panaroo_out   --clean-mode strict   --remove-invalid-genes   -t 8
```

Then build orthogroup-level expression outputs:

```bash
onebio-atlas orthogroup   --panaroo work/run1/panaroo/panaroo_out/gene_presence_absence.csv   --run-index work/run1/rnaseq/rnaseq_run_index.tsv   --genomes work/run1/genomes   --out work/run1/orthogroups   --aggregate sum
```

Outputs:

```text
locus_to_orthogroup.tsv
panaroo_locus_match_coverage.tsv
panaroo_unmatched_examples.tsv
orthogroup_tpm_long.tsv
orthogroup_expression_mapping_coverage.tsv
X_orthogroup_tpm.tsv
X_orthogroup_copy_number.tsv
orthogroup_amr_hits.tsv
```

### Same-gene/different-loci catalog

```bash
onebio-atlas same-gene   --db work/run1/atlas/atlas.sqlite   --out work/run1/same_gene_loci   --expr-threshold 1   --min-loci 2   --min-biosamples 2   --min-delta-expr 5
```

The same analysis is also available interactively in the **Explore SQLite Atlas** page under the **Same gene, different loci** tab.

### Knowledge graph visualisation

Run Module F:

```bash
onebio-atlas run   --manifest work/manifest/manifest.tsv   --work-dir work/run1   --steps kg
```

Then open:

```bash
onebio-atlas ui
```

Go to **Explore SQLite Atlas -> Knowledge graph** and select a `.dot` file.
